## 五、加分项

- 将手写 Skill 拆分出 `references/`、`scripts/` 或 `assets/`，例如来源白名单、时间解析脚本或样例输出，并说明为什么这样拆分：+3 分
- 使用平台真实 Token 统计，并与粗略估算进行对比：+3 分
- 对同一个 Skill 进行两个版本的效果对比，说明哪一版触发更准确、输出更稳定：+4 分
- 将抓取结果接入后续提示词评估、Harness 或 RAG 资料库任务中：+5 分
- 小组内交换 Skill 互测，并记录他人使用时暴露的问题：+5 分

## 六、注意事项

1. 报告和 Skill 中不得出现真实 API Key、Token、App Secret、账号密码或企业内部敏感数据。
2. 有无 Skill 对比实验应尽量使用相同任务和相同初始资料，避免人为偏置。
3. Token 估算只用于课堂观察，不作为精确计费依据。
4. AI 生成的 Skill 草稿必须经过人工校订，不得直接提交原始输出。
5. 抓取米游社、游戏官网或游戏资讯站时，只允许使用公开页面；不得绕过登录、验证码、反爬机制、付费墙或 App 私有接口。
6. 不得编写或传播盗号、刷码、脚本抢码、批量试码、灰色交易和虚假抽奖引流方法。
7. 外部 Skill 观察以理解结构和复用边界为主，不要求同学接入私人邮箱、真实云账号或生产仓库。
8. 抄袭他人 Skill、报告或对比记录将按课程规定处理。

## 七、常见问题

### Q1: 我的 Agent 平台不支持自动挂载 Skill，怎么办？

可以使用配置文件显式声明 Skill 路径，或在报告中说明当前平台限制，并用“将 Skill 内容作为 Agent 能力上下文注入”的方式完成验证。重点是证明 Agent 按 Skill 流程工作。

### Q2: 有无 Skill 的结果差异不明显，会扣分吗？

不会。只要实验设计公平、记录完整，并能分析为什么差异不明显，也属于有效结果。可能原因包括页面内容太简单、Skill 描述太泛、Agent 本身已有类似抓取流程等。

### Q3: Token 消耗应该如何统计？

优先使用平台提供的输入、输出和总 Token 统计。没有统计时，可以用同一种粗略估算方法比较两组实验，并在报告中说明估算方法的局限。

### Q4: 外部 Skill 必须真的安装并调用吗？

不强制。若课程环境允许，可以实际调用；若不方便接入账号或插件，至少应阅读一个外部 Skill 的说明，完成结构分析和复用判断。

### Q5: Skill 是否可以包含很长的参考资料？

不建议直接放进 `SKILL.md`。长资料应放入 `references/`，并在 `SKILL.md` 中说明何时读取，避免每次触发都浪费上下文。

### Q6: 如果某个页面显示了兑换码，但没有写失效时间怎么办？

应将 `valid_until` 标记为 `unknown`，同时保留原文证据片段和抓取时间，不能自行编造失效时间。

### Q7: 米游社页面需要登录或验证码怎么办？

记录为“当前来源不适合自动抓取”，改用公开可访问的官方公告、课堂指定样例页或人工提供的脱敏页面文本，不要尝试绕过限制。
