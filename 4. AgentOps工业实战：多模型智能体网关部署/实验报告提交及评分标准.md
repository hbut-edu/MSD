## 一、实验报告基本结构

### 1.1 封面信息

- **实验名称**：AgentOps工业实战：多模型智能体网关部署
- **学生姓名**、**学号**、**班级**
- **实验日期**
- **实验环境**：操作系统、Python版本、OpenClaw版本、Ollama版本、是否使用飞书与火山引擎

### 1.2 实验报告目录

建议按以下结构组织：

1. 实验概述与环境说明
2. OpenClaw 安装与网关启动

## 八、注意事项

1. **凭证脱敏**：报告中不得出现真实 API Key、App Secret、Endpoint 私密信息。
2. **独立完成**：实验报告和关键截图必须体现个人实验过程。
3. **问题如实记录**：未完全跑通时，应说明卡在哪一步、尝试过哪些解决方法。
4. **截图清晰**：截图应能看清命令、返回结果和关键配置。
5. **配置文件谨慎提交**：如提交 `config.yaml`，必须先删除或替换敏感字段。

## 九、常见扣分点

- 只截图不解释配置关系
- 把火山引擎 API Key 或飞书 App Secret 明文写进报告
- 只完成云端 Agent，未完成本地 Agent 或 Router
- 没有说明 Endpoint ID 与模型名称的区别
- 没有任何真实问题记录或实验反思
