## 文件约定

每个实验主题目录遵循统一约定：

- `instruction.md`：实验主文档，包含实验主题、目标、课程概览、环境准备、分阶段步骤、常见问题和附录。
- `实验报告提交及评分标准.md`：学生提交要求、评分维度和验收标准。
- `附件/`：代码、图片、PPT、HTML、日志、配置样例、测试脚本及其他辅助文档。

主题根目录应尽量只保留 `instruction.md`、`实验报告提交及评分标准.md` 和 `附件/`，避免学生在入口层被过多辅助文件干扰。

## 使用方式

1. 根据课程进度进入对应主题目录。
2. 优先阅读该主题的 `instruction.md`，完成环境检查和实验前置准备。
3. 按阶段完成实验任务，必要时参考 `附件/` 中的代码、图片、课件或示例文件。
4. 实验结束后，根据 `实验报告提交及评分标准.md` 整理实验报告、截图、代码仓库链接和反思总结。
5. 涉及 API Key、App Secret、Token、Endpoint 等敏感信息时，只在本地环境变量或私有配置中保存，不要写入提交材料。

## 环境概览

不同主题依赖不同工具，常见环境包括：

- Git、IDEA、VS Code、Gitee 或 GitHub。
- Python 3.10+ 或 Python 3.11，`pip` 与虚拟环境。
- Ollama 与 Qwen3.5 系列模型。
- Gradio、MCP SDK、OpenAI 兼容接口客户端。
- OpenClaw、飞书开放平台、火山引擎豆包相关配置。
- FastAPI、Uvicorn、Docker、`asyncio`、本地 RAG 与轻量级评估脚本。

具体版本、安装方式和验证命令以各主题 `instruction.md` 中的要求为准。

## 教学维护建议

- 新增主题时，建议同步创建 `instruction.md`、`实验报告提交及评分标准.md` 和 `附件/`。
- 更新模型、SDK、平台控制台或命令行工具时，应同步检查实验步骤、截图描述和 FAQ。
- 生成课件、演示网页或示例代码后，优先放入对应主题的 `附件/`。
- 发布给学生前，建议检查 Markdown 链接、代码块闭合、评分标准和安全提示是否完整。
