## 三、提交材料

1. 实验报告文档（Markdown、Word 或 PDF）
2. 源码文件：
   - `chunker.py`
   - `simple_vector.py`
   - `retriever.py`
   - `rag_answer.py`
   - `main.py`
   - `eval_retrieval.py`
   - `eval_refusal.py`
3. `docs/course_notes.txt` 或自选教学资料
4. 检索命中率运行截图
5. 至少 3 个资料外拒答测试截图

## 四、评分标准

| 评分项 | 分值 | 评分要点 |
| --- | ---: | --- |
| 资料准备与切分 | 15 | 文档片段清晰，有来源标识和 chunk id |
| 向量化实现 | 15 | 能将问题和片段转换为可比较表示 |
| Top-K 检索 | 20 | 能返回相关片段并显示相似度或排序依据 |
| RAG 回答流程 | 20 | 能基于检索上下文生成回答并列出来源 |
| 拒答机制 | 15 | 能对资料外问题给出明确拒答 |
| 评估与分析 | 10 | 能统计命中率并分析误检、漏检原因 |
| 报告表达 | 5 | 能解释 RAG 降低幻觉的工程原理 |

## 五、加分项

- 接入 `sentence-transformers` 或 ChromaDB：+5 分
- 支持多文件资料加载：+3 分
- 输出带来源引用的自然语言回答：+3 分
- 增加简单 Web UI：+4 分

## 六、注意事项

1. 不要使用包含真实隐私或敏感信息的资料。
2. 如果使用联网模型，应避免上传不可公开资料。
3. 报告中需要说明资料外问题的判断依据。
4. 代码应能在普通同学电脑上运行。

## 七、常见问题

### Q1: 检索命中率低是否一定扣很多分？

不一定。只要报告能清楚分析原因并提出改进方案，仍可体现实验价值。

### Q2: 可以使用自己的课程资料吗？

可以，但资料内容应适合公开展示，不包含个人隐私或课程敏感信息。
