Git 文件状态机模拟器

理解工作区、暂存区与版本库之间的状态流转

状态转换公式：\[S' = f(S, \text{action})\] 其中 \(S \in \{\text{Untracked, Unmodified, Modified, Staged}\}\)
未追踪 (Untracked)
Working Directory
已修改 (Modified)
Working Directory
已暂存 (Staged)
Staging Area (Index)
未修改 (Unmodified)
Repository (HEAD)

核心逻辑说明

  • git add 将新文件或修改过的文件标记为 已暂存。此时文件快照已进入索引区。
  • git commit 将所有暂存区的文件永久保存到本地仓库，文件状态回归 未修改。
  • Modify 编辑一个已被追踪的文件，其状态会自动变为 已修改。
  • New File 新创建的文件默认是 未追踪 的，Git 不会自动管理它。
