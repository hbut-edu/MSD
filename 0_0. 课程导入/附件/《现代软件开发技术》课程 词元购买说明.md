# 课程推荐

本课程推荐使用 **Coding Plan** 而非按Token计费，原因如下：

- 每日调用量在套餐额度范围内
- 需要固定预算控制

***

## 多智能体调用：Coding Plan vs 按Token计费

### 多智能体调用特点

多智能体调用通常涉及大量**内部循环调用**，一个用户请求可能触发几十甚至上百次模型调用：

- 任务拆解
- 工具调用
- 反思迭代
- 结果验证

### 计费方式对比

| 方式              | 适用场景          | 优势           | 劣势             |
| :-------------- | :------------ | :----------- | :------------- |
| **Coding Plan** | 低频使用、调用量可控    | 固定成本，预算清晰    | 多智能体调用可能快速耗尽额度 |
| **按Token计费**    | 高频使用、复杂多智能体任务 | 按需付费，调用次数不受限 | 成本不可控，高峰期可能昂贵  |

### 选型建议

**推荐 Coding Plan 的情况：**

- 每日调用量在套餐额度范围内
- 主要用于简单代码补全，少量使用多智能体的任务
- 需要固定预算控制

**推荐按Token计费的情况：**

- 频繁使用多智能体（如OpenClaw、Claude Code等）
- 单次任务涉及大量内部调用
- 需要灵活的扩展性
