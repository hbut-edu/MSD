## 四、评分标准

| 评分项 | 分值 | 评分要点 |
| --- | ---: | --- |
| 实验环境与运行完整性 | 10 | 能正确创建环境并运行主程序，文件结构清晰 |
| 样本数据设计 | 10 | 样本覆盖查询、计算、导出、多步骤等场景，包含噪声和约束 |
| 特征词提取 | 20 | 能提取动作、对象、文件名等关键特征，规则清晰可解释 |
| 意图分类与路由 | 15 | 能根据特征词输出合理意图，分类准确率较高 |
| 结构化任务单 | 20 | JSON 字段完整，能表达动作、对象、约束和输出格式 |
| 字段校验与错误解释 | 10 | 能识别字段缺失、冲突或未知意图，并给出清楚错误信息 |
| 策略对比与分析 | 10 | 能分析规则、Zero-shot、Few-shot 或 CoT 的稳定性差异 |
| 报告表达与反思 | 5 | 描述清楚，能总结意图解析对 Agent 系统的意义 |

## 五、加分项

- 增加更多业务场景并给出自动化测试：+3 分
- 使用 `jsonschema` 或 Pydantic 完成更严格的结构校验：+3 分
- 接入本地大模型并统计结构化输出成功率：+4 分

## 六、注意事项

1. 报告中不要包含真实个人信息、企业数据或真实 API Key。
2. 引用 AI 辅助生成内容时，应说明使用位置并进行人工核验。
3. 代码应能独立运行，避免依赖本机绝对路径。
4. 抄袭他人代码或报告将按课程规定处理。

## 七、常见问题

### Q1: 没有本地大模型可以完成实验吗？

可以。本实验核心是特征词工程与结构化解析，规则版程序可以独立完成。

### Q2: 样本数量越多越好吗？

样本应优先保证代表性。建议至少覆盖查询、计算、导出、多步骤和否定约束。

### Q3: 如果分类准确率不高怎么办？

报告中应说明错误样本、失败原因和改进方案。错误分析本身也是评分重点。
