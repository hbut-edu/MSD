## 四、评分标准

| 评分项 | 分值 | 评分要点 |
| --- | ---: | --- |
| 安全策略设计 | 15 | 明确目录边界、工具白名单、文件类型限制和写入限制 |
| 工具函数实现 | 10 | 原子工具职责清晰，可读、写、追加、列出文件 |
| Harness 核心逻辑 | 20 | 能统一校验工具名、路径、参数，并返回结构化结果 |
| 越权拦截能力 | 15 | 能拦截路径穿越、未知工具、危险扩展名等请求 |

| 安全 Skill 设计 | 10 | `SKILL.md` 能说明能力范围、禁止行为和 Harness 调用流程 |
| 报告表达 | 5 | 能解释 Prompt 约束与代码级约束的区别 |

## 五、加分项

## 六、注意事项

1. 不要在实验中读取真实隐私文件。
2. 不要开放真实删除、格式化、系统命令执行等危险能力。
3. 日志示例应使用模拟数据，不要包含真实 Token 或密码。
4. 报告应说明每类拦截的原因，而不仅展示截图。

## 七、常见问题

### Q1: 没有使用真实大模型会影响成绩吗？

不会。本实验重点是 Harness 的工程实现，模拟 Agent 请求即可完成核心目标。

### Q2: pytest 不是必须的吗？

不是。可以用普通 `assert` 或终端输出验证，但自动化测试更能体现工程能力。

### Q3: 为什么要保留审计日志？

审计日志能帮助定位错误、复盘越权请求，并为后续 Agent 自我修正提供反馈依据。
