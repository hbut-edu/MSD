## 四、评分标准

| 评分项 | 分值 | 评分要点 |
| --- | ---: | --- |
| 同步基线 | 10 | 能实现串行模拟调用并记录耗时 |
| 异步并发 | 20 | 能使用 `asyncio.gather` 并发执行多个任务 |
| 超时与重试 | 20 | 能处理超时、临时失败和重试次数 |
| Token 估算 | 15 | 能估算输入输出 Token 并计算总成本 |
| 限流器实现 | 20 | 能按单分钟 Token 阈值进行等待或限制 |
| 压测与分析 | 10 | 能比较同步/异步耗时和不同限流阈值效果 |
| 报告表达 | 5 | 能说明性能、成本和稳定性之间的权衡 |

## 五、加分项

- 接入真实本地模型或 OpenAI 兼容 API 并保留限流保护：+5 分
- 将结果写入 CSV 或 JSON 日志：+3 分
- 使用图表展示并发数与耗时关系：+4 分

## 六、注意事项

1. 如接入真实 API，不要在报告中展示完整 API Key。
2. 压测并发数应适度，不要影响学校网络或公共服务。
3. 成本估算可以是近似值，但计算方法需要说明。
4. 报告应包含失败案例分析，而不仅展示成功运行结果。

## 七、常见问题

### Q1: 不接真实 API 是否可以？

可以。本实验核心是异步、重试、Token 统计和限流机制。

### Q2: 如果异步耗时波动很大怎么办？

可以多运行几次取平均值，并在报告中说明波动原因。

### Q3: Token 价格应该如何设置？

可使用实验中的默认价格，也可以自行设定模拟价格，但需要保持前后一致。
